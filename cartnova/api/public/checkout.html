Checkout Sequence Flow (for Sequence Mitigation)
The checkout flow must follow this exact sequence:
1. POST /api/v2/checkout/start — initiated from cart page
2. PUT /api/v2/checkout/{id}/shipping — set shipping address
3. PUT /api/v2/checkout/{id}/payment — set payment method
4. POST /api/v2/checkout/{id}/confirm — place the order
5. GET /api/v2/checkout/{id}/status — check confirmation
Skipping steps (e.g. jumping to confirm without shipping/payment) should be blocked by sequence mitigation rules.
